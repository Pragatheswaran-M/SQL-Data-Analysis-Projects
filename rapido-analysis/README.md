# Rapido User Behavior & Ride Analytics

## Problem Statement
Analyze ride data to understand user behavior, ride frequency, and identify high-value and low-engagement users.

## Dataset Description
This dataset simulates ride-level transactions including user activity, ride distance, and usage patterns in a mobility platform.

## Key Questions Solved
- Who are the most active users based on ride frequency?
- How can users be segmented based on ride behavior?
- Which users contribute the most to total rides?
- Identify low-engagement or inactive users

## SQL Concepts Used
- Aggregations
- CASE WHEN (Segmentation)
- CTEs
- Joins
- Filtering

## Key Insights
- A small group of highly active users contributes a large portion of total rides
- Users can be segmented into low, medium, and high engagement categories
- Identified inactive users who may require re-engagement strategies

## Business Recommendations
- Focus retention efforts on high-frequency users
- Introduce incentives for medium users to increase engagement
- Target inactive users with promotional campaigns
